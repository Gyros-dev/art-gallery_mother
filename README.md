# Анна Векслер — онлайн-галерея

Сайт-портфолио художницы: гобелены ручного ткачества и текстильные коллажи.
Статический сайт (HTML/CSS/JS без сборки), работает на GitHub Pages.

Прокачанная версия исходного наброска: единый арт-дирекшен по палитре работ
(индиго, бирюза, терракота, золото, тёплый песок), анимации ткачества,
иммерсивная галерея-coverflow, лайтбокс, таймлайн выставок и адаптив.

## Страницы

| Файл | Раздел |
|---|---|
| `index.html` | Лендинг: хиро с анимацией «ткацкого станка», интро, лента избранного |
| `pages/gallery.html` | Галерея — coverflow с 3D-боковинами, фильтры, лента миниатюр, лайтбокс |
| `pages/exhibitions.html` | Выставки — таймлайн, превью сайта выставки в оверлее |
| `pages/literature.html` | Литература — публикации с просмотром PDF |
| `pages/about.html` | Художник — биография, статистика, контакты |

## Как добавить или изменить работы

**Сайт наполняется раскладкой файлов по папкам — код и JSON править не нужно.**
Подробная инструкция для художника: **[КАК ПОПОЛНЯТЬ САЙТ.md](КАК%20ПОПОЛНЯТЬ%20САЙТ.md)**.

Коротко:

| Что добавить | Куда положить |
|---|---|
| Работу (гобелен/коллаж) | картинку в `images/art/Gobelin` или `images/art/Collage` |
| Описание к работе | `.txt` с тем же именем рядом с картинкой |
| Серию (диптих/триптих) | подпапку с картинками внутри папки категории |
| Новую категорию галереи | новую папку в `images/art` (напр. `images/art/Живопись`) |
| Заметку в галерею | `.txt`/`.md` в папку категории (без одноимённой картинки) |
| Публикацию/статью | `.txt`/`.md` в `content/texts` |
| Фото с выставки | картинки в `images/exhibitions/<имя из exhibitions.json>` |
| Контакты и соцсети | `data/site.json` |

Каталог сайта (`data/gallery.json`, `data/texts.json`, `data/exhibitions-media.json`)
собирается генератором `scripts/build-gallery.mjs`. **В CI он запускается сам при
каждом пуше** — достаточно добавить файлы и запушить (или загрузить через веб-интерфейс
GitHub). Локально можно пересобрать вручную: `node scripts/build-gallery.mjs`.

> ⚠️ **Часть названий гобеленов и коллажей — черновые** (проставлены по сюжету:
> «Рыба», «Берег», «Чудесный улов», «Натюрморт с алыми лодками», «Дуэт», «Птицы»).
> Настоящее название = имя файла, так что достаточно переименовать файл в
> `images/art/Gobelin` или `images/art/Collage`.
> Контактный e-mail в `pages/about.html` — пример, впишите реальный.
> `papa_1.jpg` и `papa_2.jpg` лежат в `images/art/` и в галерею не входят —
> если это работы Анны, положите их в нужную папку категории.

## Локальный просмотр

```bash
cd art-gallery_mother
python3 -m http.server 4188
# открыть http://localhost:4188
```

## Публикация на GitHub Pages

Публикуется на `gyros-dev.github.io/art-gallery_mother`. Настройка:
**Settings → Pages → Source → GitHub Actions**. При каждом пуше workflow
[`.github/workflows/deploy.yml`](.github/workflows/deploy.yml) запускает генератор
каталога и публикует сайт. Обновление:

```bash
git add -A && git commit -m "новые работы" && git push
```

## Структура

```
index.html            — лендинг
pages/                — внутренние страницы + navbar.html (общая шапка)
css/                  — base.css (дизайн-система) + по стилю на страницу
js/
  common.js           — навбар, зерно, появления, станок, лайтбокс, site.json
  gallery.js          — coverflow, серии, тексты, фильтры, лайтбокс
  exhibitions.js      — таймлайн + фотогалереи выставок
scripts/
  build-gallery.mjs   — собирает каталог из папок (галерея, тексты, фото выставок)
data/                 — gallery.json, texts.json, exhibitions.json,
                        exhibitions-media.json, site.json (имя/контакты/соцсети)
content/texts/        — тексты и публикации (.txt/.md)
images/art/<Категория>/          — работы галереи по категориям
images/exhibitions/<Выставка>/   — фотографии с выставок
docs/                 — PDF-материалы
```
