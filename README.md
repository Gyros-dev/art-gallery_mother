# Анна Векслер — онлайн-галерея

Сайт-портфолио художницы: гобелены ручного ткачества и текстильные коллажи.
Статический сайт (HTML/CSS/JS без сборки), работает на GitHub Pages.

Прокачанная версия исходного наброска: единый арт-дирекшен по палитре работ
(индиго, бирюза, терракота, золото, тёплый песок), анимации ткачества,
иммерсивная галерея-coverflow, лайтбокс, таймлайн выставок и адаптив.

## Страницы

| Файл | Раздел |
|---|---|
| `index.html` | Лендинг: хиро с анимацией «ткацкого станка», интро, лента избранного |
| `pages/gallery.html` | Галерея — coverflow с 3D-боковинами, фильтры, лента миниатюр, лайтбокс |
| `pages/exhibitions.html` | Выставки — таймлайн, превью сайта выставки в оверлее |
| `pages/literature.html` | Литература — публикации с просмотром PDF |
| `pages/about.html` | Художник — биография, статистика, контакты |

## Как добавить или изменить работы

**Работы раскладываются по папкам — код и JSON править не нужно.**

```
images/art/
  Gobelin/                       ← гобелены
    Сан-Марко.jpg                ← отдельная работа, название = имя файла
    Благовещение.jpg
  Collage/                       ← коллажи
    Пещерная рыба.jpg
    Большой улов/                ← ПОДПАПКА = серия (диптих/триптих),
      Большой улов - 1.jpg         части показываются вместе,
      Большой улов - 2.jpg         название работы = имя подпапки
```

Правила:

- Папки `Gobelin` и `Collage` = категории (фильтры на сайте строятся автоматически).
- **Файл** прямо в папке категории — отдельная работа; **название = имя файла**
  (без расширения). Чтобы переименовать работу — переименуйте файл.
- **Подпапка** — серия из нескольких работ (диптих, триптих), которые
  показываются вместе; название серии = имя подпапки, порядок частей — по имени
  файла (удобно `… - 1`, `… - 2`).
- Необязательное описание: положите рядом текстовый файл `Название.txt`
  (для серии — `_info.txt` внутри подпапки) — его текст покажется под работой.
- Файлы вне папок `Gobelin`/`Collage` (например `papa_1.jpg` в `images/art/`)
  в галерее не показываются.

**Как работы попадают на сайт.** На живом сайте `gyros-dev.github.io` галерея
сама читает содержимое папок через GitHub API — достаточно добавить файлы и
запушить, они появятся. Локально (и как резервная копия) используется
`data/gallery.json`; чтобы обновить его вручную, запустите:

```bash
node scripts/build-gallery.mjs
```

**Выставки** — [`data/exhibitions.json`](data/exhibitions.json). Поле `status`:
`upcoming` (текущие/будущие, с меткой «Скоро») или `past` (архив).
Если указан `url`, карточка открывает сайт выставки в оверлее.

**Литература** — список в [`pages/literature.html`](pages/literature.html); PDF кладите в `docs/`.

> ⚠️ **Часть названий гобеленов и коллажей — черновые** (проставлены по сюжету:
> «Рыба», «Берег», «Чудесный улов», «Натюрморт с алыми лодками», «Дуэт», «Птицы»).
> Настоящее название = имя файла, так что достаточно переименовать файл в
> `images/art/Gobelin` или `images/art/Collage`.
> Контактный e-mail в `pages/about.html` — пример, впишите реальный.
> `papa_1.jpg` и `papa_2.jpg` лежат в `images/art/` и в галерею не входят —
> если это работы Анны, положите их в нужную папку категории.

## Локальный просмотр

```bash
cd art-gallery_mother
python3 -m http.server 4188
# открыть http://localhost:4188
```

## Публикация на GitHub Pages

Репозиторий уже опубликован на `gyros-dev.github.io/art-gallery_mother`.
Чтобы обновить сайт этой версией:

```bash
git add -A
git commit -m "Прокачанная версия галереи: арт-дирекшен и анимации"
git push
```

Публикация уже настроена: **Settings → Pages → Source → Deploy from a branch**,
ветка `main`, папка `/` (root). GitHub сам пересобирает сайт при каждом пуше —
сборка не нужна, сайт статический.

## Структура

```
index.html            — лендинг
pages/                — внутренние страницы + navbar.html (общая шапка)
css/                  — base.css (дизайн-система) + по стилю на страницу
js/
  common.js           — навбар, зерно, появления, канвас-«станок»
  gallery.js          — coverflow, серии, фильтры, лайтбокс
  exhibitions.js      — таймлайн выставок
scripts/
  build-gallery.mjs   — собирает data/gallery.json из папок images/art/
data/                 — gallery.json (каталог работ), exhibitions.json
images/art/Gobelin, images/art/Collage — работы по категориям
docs/                 — PDF-материалы
```
