# Как пополнять сайт

Весь сайт наполняется **раскладкой файлов по папкам**. Никакого кода. Положили
файл в нужную папку — после публикации он появился на сайте в нужном разделе.

Есть два способа добавлять файлы:

- **Через сайт GitHub** (проще всего, ничего устанавливать не нужно): открываете
  папку на github.com, перетаскиваете туда файлы, нажимаете «Commit changes».
  Через 1–2 минуты сайт обновляется сам.
- **С компьютера** (если удобнее): кладёте файлы в папки проекта и делаете
  `git add -A && git commit -m "новые работы" && git push`.

После загрузки сборка сайта запускается автоматически (GitHub Actions) — каталог
пересобирается сам, ничего вручную запускать не нужно.

## 1. Добавить работу в галерею

Папки категорий: `images/art/Gobelin` (гобелены) и `images/art/Collage` (коллажи).

- **Одна работа** — положите картинку в папку категории.
  Название работы = имя файла. Пример: `images/art/Gobelin/Ангел.jpg` → работа «Ангел».
- **Описание работы** (материал, размер, год) — положите рядом текстовый файл
  с **тем же именем**. Пример: `images/art/Gobelin/Ангел.txt`:
  ```
  Шерсть, ручное ткачество, 60 × 80 см, 2016
  ```
  Этот текст покажется под работой и в увеличении.
- **Серия** (диптих, триптих, несколько связанных работ) — создайте **подпапку**,
  название серии = имя подпапки, части внутри назовите по порядку:
  ```
  images/art/Collage/Времена года/
      Времена года - 1.jpg
      Времена года - 2.jpg
      Времена года - 3.jpg
  ```
  На сайте они покажутся вместе, а в увеличении листаются стрелками.

## 2. Добавить новую категорию галереи

Просто создайте новую папку внутри `images/art`, например `images/art/Живопись`,
и кладите туда картинки. На сайте автоматически появится новый фильтр «Живопись».
(Для латинских имён можно задать русскую подпись в `scripts/build-gallery.mjs`,
поле `CATEGORY_LABELS` — но это необязательно.)

## 3. Добавить текст / публикацию

- **Короткая заметка прямо в категорию галереи** — положите `.txt`/`.md` в папку
  категории (без картинки с таким же именем). Первая строка — заголовок, дальше —
  текст. Он покажется в галерее отдельной карточкой.
- **Статья / эссе в раздел «Литература → Публикации»** — положите `.txt`/`.md`
  в папку `content/texts`. Первая строка — заголовок, остальное — текст.
- **PDF-пособие** — положите файл в `docs/` и добавьте строку в список на
  странице `pages/literature.html`.

## 4. Добавить фотографии с выставки

1. В `data/exhibitions.json` у нужной выставки есть поле `"photos"` — это имя папки.
2. Создайте папку `images/exhibitions/<это имя>` и положите туда фотографии.
   Например для `"photos": "Триеннале 2025"` → `images/exhibitions/Триеннале 2025/`.
3. На странице «Выставки» у этой выставки появятся миниатюры и кнопка
   «Фотографии», открывающая галерею.

Чтобы добавить **новую выставку**, допишите блок в `data/exhibitions.json`
(скопируйте существующий и поменяйте текст). Поля: `title`, `date`, `location`,
`description`, `url` (сайт выставки, необязательно), `photos` (папка с фото,
необязательно), `status` — `upcoming` (текущая/будущая) или `past` (архив).

## 5. Имя, контакты, соцсети

Всё — в файле `data/site.json`: имя, e-mail, телефон, ссылки на ВКонтакте,
Telegram, WhatsApp, Instagram. Заполненные ссылки появятся в подвале сайта и в
разделе «Художник». Пустые — просто не показываются.

### Коротко

| Что добавить | Куда положить |
|---|---|
| Работу (гобелен/коллаж) | картинку в `images/art/Gobelin` или `images/art/Collage` |
| Описание к работе | `.txt` с тем же именем рядом с картинкой |
| Серию (диптих/триптих) | подпапку с картинками внутри папки категории |
| Новую категорию | новую папку в `images/art` |
| Заметку в галерею | `.txt`/`.md` в папку категории |
| Публикацию/статью | `.txt`/`.md` в `content/texts` |
| Фото с выставки | картинки в `images/exhibitions/<имя из exhibitions.json>` |
| Контакты и соцсети | `data/site.json` |
